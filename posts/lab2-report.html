实验二：基于 GitHub Pages 的个人博客搭建与报告发布

本实验以 linzy1011.github.io 仓库为载体，完成个人博客首页搭建、实验报告博文化发布、 本地预览验证以及 GitHub Pages 部署准备，最终形成一个可持续扩展的静态博客站点。

作者 LinZY1011
仓库 linzy1011.github.io
实验日期 2026-03-27
目标地址 https://linzy1011.github.io/

一、实验目标

本实验的目标是构建一个可公开访问的个人博客或个人网站，并将实验报告本身作为一篇博文发布到站点中。 为了降低部署复杂度、提高复现性，本次采用 GitHub Pages 提供的静态托管能力， 将站点直接部署到与用户名同名的主页仓库中。

二、环境构建

本地开发环境基于 Windows，实际检查到的工具版本如下：

  • Git：2.44.0.windows.1
  • Python：3.12.4
  • Node.js：v20.11.1

其中 Git 用于版本管理与代码推送，Python 用于通过 http.server 快速预览静态站点， Node.js 作为常见前端环境保留，便于后续继续扩展自动化构建或静态站点生成能力。

三、方案选择、适用范围与局限

本实验采用纯静态 HTML + CSS 方案，不依赖数据库、服务端框架或额外构建链。 其核心思路是把首页和文章页直接放在仓库根目录与 posts/ 目录中， 使默认分支中的文件能够被 GitHub Pages 直接托管。

  • 适用范围：课程实验、个人主页、作品集、轻量博客等内容更新频率不高的站点。
  • 基本假设：仓库名与 GitHub 用户名同名，默认分支为 main，并具备正常推送权限。
  • 主要局限：文章新增需要手动维护页面，不具备评论、搜索和后台管理功能。

四、站点结构设计

为了让博客可以直接发布，本次采用如下目录结构：

linzy1011.github.io/
├─ index.html
├─ styles.css
├─ README.md
├─ .nojekyll
└─ posts/
   └─ lab2-report.html

其中 index.html 作为博客首页，负责展示站点介绍与文章入口； posts/lab2-report.html 作为实验报告博文； styles.css 提供统一视觉风格和响应式布局； .nojekyll 用于确保 Pages 以静态文件方式处理站点内容。

五、实现过程

  • 在仓库根目录新建首页，设计站点介绍、实验概览、发布步骤和文章入口。
  • 将实验报告整理为一篇独立文章，放置在 posts/lab2-report.html 中。
  • 使用统一 CSS 变量和响应式布局，保证桌面端与移动端浏览体验一致。
  • 补充 README，说明本地预览、推送发布和 GitHub Pages 注意事项。

这种实现方式遵循“小步可审查”的原则：页面职责明确，文件数量少，后续继续添加文章时， 只需要在 posts/ 目录增加新页面并在首页补上入口即可。

六、本地验证

本地预览采用 Python 内置 HTTP 服务，命令如下：

python -m http.server 8080

验证重点包括：

  • 首页是否能够正常打开，导航链接是否可达。
  • 实验报告页面是否可以从首页跳转进入。
  • 样式是否在桌面端与窄屏设备下保持可读性。

七、GitHub Pages 发布流程

由于仓库名已经是 linzy1011.github.io，它天然对应 GitHub Pages 用户主页仓库。 因此只需将站点文件提交到默认分支，即可使用 GitHub Pages 提供的静态托管能力。 

git add index.html styles.css posts/lab2-report.html README.md .nojekyll
git commit -m "feat: launch personal blog on GitHub Pages"
git push origin main

推送完成后，通常可以直接通过 https://linzy1011.github.io/ 访问站点。 若仓库没有自动发布，可在仓库的 Settings -> Pages 中确认发布源指向默认分支根目录。

八、结果验证与合理性检查

完成部署后，应当至少检查以下结果是否成立：

  • 主页地址可以打开，并显示博客首页内容。
  • 点击“实验报告”后，能够进入本篇博文。
  • 文章内容与实验要求一致，确实描述了环境构建、站点搭建和发布过程。
  • 页面在手机宽度下未出现明显布局错乱。

从工程直觉上看，静态站点文件越少、层级越浅，GitHub Pages 的发布链路越稳定； 本次方案符合这一点，因此适合作为实验作业的实现基础。

九、总结与后续改进

本实验完成了从环境确认、站点搭建、本地验证到 GitHub Pages 发布准备的完整闭环， 并将实验报告本身作为博文发布到了博客中。该方案实现成本低、可复现性强， 非常适合作为个人博客的第一版。

后续如果需要继续完善，可以引入 Markdown 自动生成文章、标签分类、归档页面或自定义域名， 逐步把课程实验博客演进为长期维护的个人技术网站。
