课程实验报告

实验二：基于 GitHub Pages 的个人博客搭建与报告发布

本实验使用 linzy1011.github.io 仓库搭建一个简洁的个人博客， 并把实验报告整理成站点中的一篇文章进行发布。

作者 LinZY1011
仓库 linzy1011.github.io
实验日期 2026-03-27
目标地址 https://linzy1011.github.io/

一、实验目标

完成个人博客主页的搭建，并将实验报告作为一篇博文发布到站点中。 本次选择 GitHub Pages 作为托管平台，原因是部署简单、适合静态页面，也方便课程实验快速完成。

二、环境构建

本地开发环境基于 Windows，使用到的核心工具如下：

  • Git：2.44.0.windows.1
  • Python：3.12.4
  • Node.js：v20.11.1

Git 用于版本管理和推送代码，Python 用于本地启动静态服务预览页面， Node.js 作为前端常用环境保留，便于后续继续扩展。

三、实现过程

整个博客采用纯静态 HTML + CSS 实现。首页只保留个人简介和实验报告入口， 报告页单独放在 posts/ 目录下，以便形成清晰的结构。

  • 在仓库根目录创建 index.html 作为主页。
  • 在 posts/lab2-report.html 中编写实验报告内容。
  • 使用统一样式文件 styles.css 完成页面排版和响应式适配。
  • 通过 GitHub Pages 发布默认分支中的站点内容。

这种方案的优点是结构简单、部署直接，但局限也很明显： 如果后续文章变多，就需要手动维护更多页面。

四、本地验证与发布

本地预览采用 Python 内置 HTTP 服务，命令如下：

python -m http.server 8080

启动后可以在浏览器中检查首页和实验报告页是否都能正常访问。 确认无误后，将文件推送到仓库默认分支即可完成 GitHub Pages 发布。 

git add index.html styles.css posts/lab2-report.html README.md .nojekyll
git commit -m "feat: launch personal blog on GitHub Pages"
git push origin main

五、实验结果与总结

本实验完成了个人博客主页搭建，并成功将实验报告发布为博客文章。 最终站点内容保持简洁，只保留主页简介和实验报告，满足实验要求，也便于后续继续维护。

这一方案适合课程实验和个人主页场景；如果后续内容增多，可以再考虑引入更系统的博客框架。
