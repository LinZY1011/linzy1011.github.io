实验二：利用云平台搭建个人博客

本文对应《云计算技术》课程实验二，目标是完成个人博客/网站构建，并将实验报告内容以博文形式发布。

一、实验目的

通过实际操作掌握网站部署的基本流程，完成环境构建、博客页面搭建、报告撰写与发布，形成可运行、可复现的实验成果。

二、硬件与软件环境

  • 硬件环境：联网 PC 一台。
  • 操作系统：Windows。
  • 开发环境：Node.js v20、Python 3。
  • 网站形态：静态博客（HTML + CSS）。

三、实验步骤与内容

  • 步骤 1：完成环境构建，确认 Windows 环境下 Node.js 与 Python 可用，作为页面开发与本地调试基础。
  • 步骤 2：创建 `blog-site/` 目录并规划结构，采用“首页 + 文章页 + 样式 + 部署配置”分层组织。
  • 步骤 3：编写 `index.html`，实现博客入口、实验简介和文章跳转，形成站点首页。
  • 步骤 4：编写 `posts/lab2-report.html`，将实验报告整理为网页正文，满足“报告即博文”的发布要求。
  • 步骤 5：编写 `styles.css`，实现响应式布局、统一配色和动画效果，保证桌面端与移动端可读性。
  • 步骤 6：本地运行 `python -m http.server 8080`，访问首页和博文页检查链接与样式，确认功能可用。

四、网站构建说明

网站采用静态博客方案，不依赖数据库。核心设计思路是将页面内容和部署配置分离：页面文件负责展示，Nginx 配置负责上线。 这种架构开发成本低，部署链路短，适合课程实验和个人技术展示。

  • 页面层：`index.html` 提供导航与文章入口，`posts/lab2-report.html` 承载实验报告正文。
  • 样式层：`styles.css` 统一字体、颜色与间距，避免页面风格不一致。
  • 部署层：`deploy/nginx-site.conf` 提供站点根目录与路由规则，保证上线行为与本地一致。

五、服务器部署方案

部署目标为 Linux 云服务器（如阿里云 ECS、腾讯云 CVM）。将 `blog-site` 上传到服务器后，使用 Nginx 托管静态目录。 

# 1) 上传站点目录到服务器
scp -r blog-site user@SERVER_IP:/var/www/personal-blog

# 2) 安装 Nginx（Ubuntu）
sudo apt update
sudo apt install -y nginx

# 3) 启用站点配置
sudo cp /var/www/personal-blog/deploy/nginx-site.conf /etc/nginx/sites-available/personal-blog.conf
sudo ln -s /etc/nginx/sites-available/personal-blog.conf /etc/nginx/sites-enabled/personal-blog.conf
sudo nginx -t
sudo systemctl reload nginx

完成后可通过服务器公网 IP 访问首页；若防火墙启用，请放行 80/443 端口。

六、域名接入与 HTTPS

域名部署分为 DNS 解析和证书配置两个阶段。先在域名服务商控制台添加 A 记录指向服务器公网 IP，再在服务器上签发 HTTPS 证书。 

# 1) 安装 certbot
sudo apt install -y certbot python3-certbot-nginx

# 2) 自动配置 HTTPS（替换为真实域名）
sudo certbot --nginx -d blog.example.com -d www.blog.example.com

# 3) 验证证书自动续期
sudo certbot renew --dry-run

证书生效后，博客可通过 `https://` 安全访问，满足公网发布与后续长期维护需求。

七、发布结果

报告博文发布路径为 /posts/lab2-report.html。访问首页后点击“阅读全文”即可进入本实验报告页面，满足“将报告作为博文发布”的要求。

八、结论分析与体会

本次实验完成了从页面开发、本地验证、服务器部署到域名接入的完整闭环。实践结果表明，静态博客方案具备实现快、维护成本低、上线路径清晰的优势。后续可接入 CI/CD 自动发布、日志监控和访问统计，进一步提升工程化水平。
